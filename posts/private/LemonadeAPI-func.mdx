## 1. Initializing Lemonade

![1](/img/2017-7-20-LemonadeAPI-func/35.png)

Declare the following in the very first scene of your project.

### 1-1. Lemonade.init()

Initialization process. Lemonade.isInitialized becomes true upon successful initialization.

![1](/img/2017-7-20-LemonadeAPI-func/36.png)

![1](/img/2017-7-20-LemonadeAPI-func/37.png)

> [Reference](https://github.com/LemonClub/lemonade-android-api-unity-example/blob/master/Assets/ExGame.cs#L11)

### 1-2. Lemonade.Login()

Login process. It is essential to call this function when Lemonade.isInitialized == true.

> Connect existing users or new users to the game upon initialization using Lemonade.init. Pass Lemonade.Login() as the first parameter when successful. ([Reference](https://github.com/LemonClub/lemonade-android-api-unity-example/blob/master/Assets/ExGame.cs#L14))

## 3. Custom Database

How to use a custom DB defined by the user.

To use this:

![1](/img/2017-7-20-LemonadeAPI-func/43.PNG)

The Access permission on the Lemonade developer [site](http://lemontree.dothome.co.kr/lemonade/) must be at **level 2**.

### 3-0. Before Using

![1](/img/2017-7-20-LemonadeAPI-func/42.PNG)

Added SKILL and INVEN to the Developer Setting Table. (Not essential as they are created for demonstration purposes.)

> Note: The TOKEN column is a column designed to prevent duplicate entries in the table.

### 3-1. Adding Custom Database

![1](/img/2017-7-20-LemonadeAPI-func/44.PNG)
