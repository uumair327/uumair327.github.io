## Introduction

Chapter 1 covers the basic setup required in Unity before applying Lemonade.

## References

- API Applied Unity Sample: [Link](https://github.com/LemonClub/lemonade-android-api-unity-example)
- Lemonade Developer [Site](http://lemontree.dothome.co.kr/lemonade/)

## 1. Running Unity

![1](/img/2017-7-20-LemonadeAPI-func/1.png)

Start by creating a Unity project.

## 2. Applying the package

![3](/img/2017-7-20-LemonadeAPI-func/3.PNG)

Download the latest version from the [Release Site](https://github.com/LemonClub/lemonade-android-api-unity-example/releases) and import it into Unity.

![4](/img/2017-7-20-LemonadeAPI-func//4.PNG)

## 3. Creating AndroidManifest

> Reference: [Unity-Plugin-01.Unity Setup](/lib/2017/03/23/Unity-Plugin-01)

<br />

### 3-1 Setting the package name

![1](/img/2017-7-20-LemonadeAPI-func/6.PNG)

Set the package name to 'com.game.test'.

> Be careful to set this package name as a priority for future integrations.

<br />

### 3-2 Building the APK

![1](/img/2017-7-20-LemonadeAPI-func/7.PNG)

Next, go to Build Settings/Android/Build. You can also Build and Run.

<br />

### 3-3 Finding AndroidManifest

![1](/img/2017-7-20-LemonadeAPI-func/8.PNG)

If you have built the APK, you will find AndroidManifest.xml in the project/Temp/StagingArea/ directory.

![1](/img/2017-7-20-LemonadeAPI-func/9.PNG)

Copy or move it to project/Assets/Plugins/Android/.

## 4. Creating the \_Lemonade Object

![1](/img/2017-7-20-LemonadeAPI-func/34.PNG)

Create a GameObject named '\_Lemonade' and add the Lemonade script inside.

> The object's name must be '\_Lemonade', so be careful.

The initial setup required in Unity is now complete. Next, the work in Android Studio will finalize the settings required to use Lemonade.
